# Colab Voice TTS

Для нового workflow нужен только один notebook:

`voice_tts_colab_gpu.ipynb`

Он выполняет весь цикл:

1. Проверяет hosted Google Colab и L4.
2. Получает выбранный Git-коммит и вычисляет его точный GHCR-тег `sha-<12 символов>`.
3. Подключает Google Drive, а при сбое DriveFS продолжает во временном `/content/voice-tts-data`.
4. В основном режиме скачивает публичный OCI/Docker-образ из GHCR, показывает процент, оставшийся объём, скорость и ETA, запускает образ через `udocker` и подключает NVIDIA-библиотеки L4.
5. Проверяет CUDA внутри образа; в резервном режиме создаёт изолированное Python 3.10 окружение.
6. Устанавливает VoxCPM2 в отдельное окружение, сохраняет его веса на Drive и прогревает localhost-worker.
7. Хранит веса CosyVoice 3 и VoxCPM2 в `MyDrive/Voice TTS/models/`.
8. Запускает Gradio с переключателем **VoxCPM2 2B / CosyVoice 3**.
9. Сохраняет библиотеку голосов в `MyDrive/Voice TTS/voices/`.
10. Сохраняет результаты и выбранный движок в `MyDrive/Voice TTS/runs/`.
11. Останавливает приложение, VoxCPM2-worker и временную ссылку.

Hosted Colab не запускает Docker daemon. `udocker` исполняет доверенный контейнерный образ в user space без полноценной Docker-изоляции. Перед запуском дождитесь GitHub Actions, сделайте GHCR package публичным, оставьте готовые `REPO_URL` и `EXECUTION_MODE = "udocker"`, затем выполните notebook сверху вниз. Если этот runtime несовместим с `udocker`, смените режим на `native`. Для остановки поставьте `STOP_APP = True` и отдельно выполните финальную ячейку.

Долгая ячейка подготовки показывает номер этапа и сколько этапов осталось. Если внешняя команда временно не печатает собственный вывод, notebook каждые 15 секунд сообщает прошедшее время и подтверждает, что PID процесса активен. Отдельно виден статус VoxCPM2 (`starting`, `loading`, `ready`) и оставшееся время до 30-минутного таймаута. При повторном запуске в том же Colab runtime уже подготовленные окружения используются повторно. Запуск Gradio также печатает heartbeat каждые 10 секунд и оставшееся время до явного таймаута.

Предыдущие раздельные setup/workflow notebooks удалены: весь актуальный запуск, включая необязательный импорт `local_voice_library_upload.zip`, находится в одном файле.

Полная инструкция, GitHub/GHCR и Docker-схема описаны в корневом `README.md`.
